# Leitfaden: Website-Erstellung als System
### Übertragbares Praxishandbuch für professionelle Websites

*Erstellt auf Basis des Projekts nadinejoest.de · Juni 2026*

---

## 1. Grundprinzipien der Website-Erstellung

### Denkweise

Eine Website ist kein digitales Visitenkärtchen — sie ist ein Verkaufsgespräch, das rund um die Uhr läuft. Jede Entscheidung, von der Struktur bis zur Farbe, dient einem einzigen Ziel: den Besucher zur gewünschten Handlung zu führen.

**Die drei Kernfragen hinter jeder Entscheidung:**
1. Hilft das dem Besucher, schnell zu verstehen, was er hier bekommt?
2. Führt das den Besucher einen Schritt näher zur Kontaktaufnahme?
3. Schafft das Vertrauen?

### Strategische Grundentscheidungen (immer zuerst)

Bevor eine einzige Zeile Code geschrieben wird, müssen folgende Punkte feststehen:

- Wer ist die Zielgruppe — genau, nicht grob?
- Was ist das eine Hauptziel der Website (Anfragen generieren, Termine buchen, Produkte verkaufen)?
- Welche eine Handlung soll der Besucher ausführen?
- Was unterscheidet diese Person/dieses Angebot von anderen?

### Typische Erfolgsfaktoren

| Faktor | Warum wichtig |
|---|---|
| Klare Positionierung | Besucher entscheiden in 3–5 Sekunden, ob sie bleiben |
| Mobil-First-Design | 60–70 % der Besucher kommen vom Smartphone |
| Schnelle Ladezeit | Jede Sekunde kostet Besucher |
| Ein klarer CTA pro Seite | Zu viele Optionen = keine Entscheidung |
| Vertrauenselemente | Ohne Vertrauen keine Anfrage |

## 2. Website-Strategie (vor der Umsetzung)

### Zieldefinition

Das Hauptziel muss in einem Satz formulierbar sein:

> „Die Website soll \[Zielgruppe\] dazu bringen, \[konkrete Handlung\] auszuführen."

**Beispiel nadinejoest.de:**
> „Die Website soll selbstständige Coaches und Unternehmerinnen dazu bringen, ein kostenloses Erstgespräch zu buchen."

### Fragebogen vor jedem Projekt

Diese Fragen müssen vor Projektstart vollständig beantwortet sein:

**Zum Angebot:**
- Was wird angeboten? (Konkret, nicht allgemein)
- Was sind die 2–3 größten Vorteile für die Kundin?
- Gibt es Pakete/Preise, die kommuniziert werden sollen?
- Was ist das Alleinstellungsmerkmal?

**Zur Zielgruppe:**
- Wer ist die Wunschkundin? (Branche, Situation, Herausforderung)
- Welche Sprache spricht sie? (Fachbegriffe vs. Umgangssprache)
- Welches Problem hat sie, das gelöst wird?
- Was hält sie davon ab, sofort zu kaufen?

**Zum Unternehmen:**
- Wie lautet der vollständige Name für das Impressum?
- Welche Adresse, Telefonnummer, E-Mail wird verwendet?
- Gibt es bereits ein Logo, Farben, eine Schriftart?
- Welche Domain soll verwendet werden?

**Technisch:**
- Wo wird die Website gehostet?
- Wer verwaltet die Domain?
- Gibt es ein Kontaktformular — wenn ja, welcher Dienst?
- Soll ein Buchungssystem integriert werden?

### Positionierung formulieren

**Formel:**
> „Ich helfe \[Zielgruppe\] dabei, \[Problem\] zu lösen, damit \[gewünschtes Ergebnis\]."

Diese Formel wird zum Fundament aller Website-Texte.

## 3. Website-Struktur (Blueprint)

### Standard-Seitenstruktur für Dienstleister

```
Startseite        → Überblick + erster Eindruck + Hauptkonversion
Leistungen        → Angebot im Detail + Pakete + Preise
Über mich         → Vertrauen + Persönlichkeit + Identifikation
Kontakt           → Konversion + Formular + Buchungsoption
Impressum         → Pflicht (rechtlich)
Datenschutz       → Pflicht (rechtlich)
```

### Zweck jeder Seite

**Startseite**
Beantwortet in 5 Sekunden: Für wen? Was? Warum hierher?
Enthält: Hero-Bereich, Problem-Darstellung, Lösung/USPs, CTA

**Leistungen**
Beantwortet: Was genau, zu welchem Preis, wie läuft das ab?
Enthält: Leistungsbeschreibungen, Pakete, Prozessschritte, CTA

**Über mich**
Beantwortet: Warum sollte ich dieser Person vertrauen?
Enthält: Persönliche Geschichte, Werdegang, Werte, Foto, Für-wen-Abschnitt

**Kontakt**
Einziges Ziel: Kontakt herstellen.
Enthält: Formular, Buchungslink, Kontaktdaten, kein Ablenkung

### Logische Nutzerführung (Conversion Flow)

```
Startseite
  → Problem erkannt? → Leistungen
  → Wer steckt dahinter? → Über mich
  → Überzeugt? → Kontakt / Erstgespräch buchen
```

Jede Seite endet mit einem klaren Hinweis, wohin als nächstes.

### Wann die Struktur angepasst wird

- **Produkte verkaufen** → Shop-Seite ergänzen
- **Mehrere Zielgruppen** → Trennung der Leistungsseiten
- **Blog/Content-Marketing** → Blogbereich ergänzen (erst ab Phase 2)
- **Agentur/Team** → Team-Seite ergänzen
- **Portfolio nötig** → Portfolio-Seite ergänzen

## 4. Content-System (Texte & Messaging)

### Grundprinzip

Gute Website-Texte sprechen die Sprache der Zielgruppe, benennen das Problem bevor die Lösung kommt, und machen den Nutzen konkret — nicht das Feature.

**Feature vs. Nutzen:**
- Feature: „Ich übernehme dein E-Mail-Management"
- Nutzen: „Deine Inbox ist leer, bevor du morgens den Laptop aufklappst"

### Aufbau einer überzeugenden Seite (AIDA-Prinzip)

```
A — Attention:   Headline, die das Problem oder den Wunsch trifft
I — Interest:    Zeige, dass du das Problem verstehst
D — Desire:      Zeige die Lösung und ihre Vorteile
A — Action:      Klarer, einziger Call-to-Action
```

### Headline-Formeln

**Problem-Formel:**
> „Du liebst dein Business — aber \[konkretes Problem\] raubt dir die Energie."

**Ergebnis-Formel:**
> „Damit du dich auf \[Wunschergebnis\] konzentrieren kannst."

**Identitäts-Formel:**
> „Nicht \[allgemeine Alternative\]. Sondern \[Positionierung\]."

**Persönliche Formel:**
> „Hi, ich bin \[Name\], und ich \[was du tust + für wen\]."

### CTA-Formulierungen (wiederverwendbar)

| Kontext | Formulierung |
|---|---|
| Hauptaktion | „Kostenloses Erstgespräch buchen" |
| Weicher Einstieg | „Unverbindlich anfragen" |
| Mehr erfahren | „Leistungen ansehen" |
| Kontakt | „Nachricht schreiben" |
| Vertrauensaufbau | „Lern mich kennen" |

**Regel:** Nie mehr als zwei CTAs pro Seitenabschnitt. Einer primär (ausgefüllt), einer sekundär (nur Rahmen).

### Tonalität definieren

Vor jedem Projekt festlegen:
- Du oder Sie?
- Professionell-distanziert oder nahbar-persönlich?
- Kurze Sätze oder ausführliche Erklärungen?
- Emotionale oder sachliche Sprache?

**Beispiel nadinejoest.de:** Du, nahbar-professionell, kurze prägnante Sätze, emotionale Ansprache.

### Texte auf Zeichenanzahl prüfen

| Element | Empfehlung |
|---|---|
| Meta-Titel | 50–60 Zeichen |
| Meta-Description | 140–160 Zeichen |
| H1 pro Seite | Genau eine, enthält Hauptkeyword |
| Hero-Untertitel | 2–3 Sätze, maximal |

## 5. Design- & UX-Prinzipien

### Design-System einrichten (einmalig, dann wiederverwenden)

Zu Beginn jedes Projekts festlegen:

```css
Primärfarbe:     z. B. #8C9795 (Markenfarbe)
Hintergrundton:  z. B. #F5F0E8 (Beige/Warm)
Textfarbe:       z. B. #2D2D2D (Dunkelgrau, kein reines Schwarz)
Schriftarten:    1 Serifenschrift (Überschriften) + 1 serifenlose (Fließtext)
Abstände:        Einheitliches Raster (z. B. 8px-System)
Border-Radius:   Einheitlich (z. B. 8px oder 12px)
```

### Schriftarten-Kombination (bewährt)

| Kombination | Wirkung |
|---|---|
| Cormorant Garamond + DM Sans | Elegant, weiblich, modern |
| Playfair Display + Inter | Hochwertig, neutral |
| Lora + Source Sans Pro | Warm, vertrauenswürdig |

Quelle: Google Fonts (kostenlos, via @import in CSS)

### Mobile-First-Regeln

- Navigation: Hamburger-Menü ab 768px (oder 640px)
- Schriftgrößen: Minimum 16px Fließtext
- Buttons: Minimum 44px Tipp-Fläche
- Abstände: Padding auf Mobile mindestens 20px seitlich
- Bilder: `max-width: 100%` immer gesetzt
- Grids: Auf Mobile immer einspaltiger Stack

**Häufiger Fehler:** `grid-template-columns: repeat(3, 1fr)` als Inline-Style setzt das CSS-Media-Query außer Kraft. Immer Grid-Layouts nur über CSS-Klassen steuern.

### Conversion-orientiertes Design

- **Kontrast:** CTA-Buttons müssen sich klar vom Hintergrund abheben
- **Weißraum:** Mehr Abstand = höhere Wahrnehmung von Qualität
- **Fotos:** Echte Fotos der Person schaffen mehr Vertrauen als Stock-Bilder
- **Hierarchie:** H1 → H2 → H3, nie überspringen
- **Konsistenz:** Gleiche Komponenten auf allen Seiten (Nav, Footer, Buttons)

### Häufige Designfehler

- Zu viele Schriftarten (max. 2)
- Zu viele verschiedene Farben (max. 3–4)
- Text direkt auf Bild ohne ausreichenden Kontrast
- `min-height: 100vh` im Hero — führt zu leerem Raum auf Desktop
- Inline-Styles statt CSS-Klassen (schwer zu warten)

## 6. SEO-Grundsystem

### Für jede Seite verpflichtend

```html
<title>[Seitenthema] | [Name] – [Positionierung]</title>
<meta name="description" content="[Konkreter Nutzen, 140–160 Zeichen]" />
<meta name="robots" content="index, follow" />
```

### Open Graph (für Social Sharing)

```html
<meta property="og:title" content="..." />
<meta property="og:description" content="..." />
<meta property="og:type" content="website" />
<meta property="og:locale" content="de_DE" />
<meta property="og:image" content="https://domain.de/og-bild.jpg" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
```

**OG-Bild:** 1200×630px, als JPG, enthält Name + Kernaussage + ggf. Foto.
Empfehlung: In Canva erstellen (kostenlos, kein Account für Download nötig wenn direkt gespeichert).

### Favicon

```html
<link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png" />
```

Erstellen unter: favicon.io → Text to ICO → Initialen, Markenfarbe, Download.

### Keyword-Logik

- **1 Haupt-Keyword pro Seite** (im Titel, H1, erster Absatz)
- **2–3 Neben-Keywords** (in Überschriften und Fließtext)
- Keine Keyword-Häufung — natürliche Sprache hat Vorrang

**Beispiel:**
- Startseite: „Business Support" (Haupt) + „Virtuelle Assistenz", „Backoffice" (Neben)
- Leistungen: „E-Mail-Management" + „Prozessoptimierung", „Controlling"

### Sitemap

Datei `sitemap.xml` im Hauptordner:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://domain.de/</loc>
    <lastmod>YYYY-MM-DD</lastmod>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
  </url>
  <!-- weitere Seiten analog -->
</urlset>
```

Prioritäten: Startseite 1.0 → Hauptseiten 0.8–0.9 → Rechtliches 0.3

Nach Livegang in **Google Search Console** einreichen.

### Google Search Console (Einmalig einrichten)

1. search.google.com/search-console aufrufen
2. Domain verifizieren (HTML-Datei-Methode: Datei herunterladen, hochladen, verifizieren)
3. Sitemap einreichen unter „Sitemaps"
4. Nach 2–4 Wochen: Berichte prüfen

### robots.txt (einfachste Version)

```
User-agent: *
Allow: /
Sitemap: https://domain.de/sitemap.xml
```

## 7. Conversion-Logik

### Wie Besucher zu Anfragen werden

```
Aufmerksamkeit → Interesse → Vertrauen → Handlung
```

Jede dieser Phasen braucht eigene Elemente auf der Website.

### Vertrauenselemente (nach Wirkung sortiert)

1. **Echte Fotos** der Person (kein Stock)
2. **Kundenstimmen / Testimonials** (stärkster sozialer Beweis)
3. **Konkreter Werdegang** mit Zahlen ("13 Jahre Erfahrung")
4. **Klarer Prozess** (was passiert nach der Anfrage?)
5. **Transparente Preise** (nimmt Unsicherheit)
6. **Vollständiges Impressum** (rechtliche Seriosität)
7. **Professionelles Design** (Qualitätssignal)

### CTA-Platzierung (Regel)

- **Hero:** Primärer CTA (Hauptziel der Website)
- **Nach Problem-Darstellung:** Sekundärer CTA
- **Ende jedes Abschnitts:** Weiterführender Link
- **Footer:** Alle wichtigen Links
- **Feste Navigation:** CTA-Button immer sichtbar

### Psychologische Grundlagen (einfach erklärt)

**Klarheit vor Kreativität**
Besucher wollen sofort verstehen, was sie bekommen. Clevere Formulierungen, die erst erklärt werden müssen, kosten Konversionen.

**Spezifität schafft Vertrauen**
„Ich helfe Unternehmerinnen" ist schwach. „Ich helfe selbstständigen Coaches, ihre Inbox auf null zu bringen" ist stark.

**Reziprozität**
Etwas kostenlos geben (Erstgespräch, Tipps) erhöht die Bereitschaft zu kaufen.

**Social Proof**
Menschen folgen anderen. Testimonials, Bewertungen und konkrete Fallbeispiele sind die wirksamsten Überzeugungsmittel.

## 8. Technische Umsetzung (Systematisiert)

### Empfohlenes Setup für einfache Websites (ohne CMS)

| Bereich | Tool | Kosten |
|---|---|---|
| Domain | IONOS, Namecheap, Hetzner | ~10–15 €/Jahr |
| Hosting (statisch) | GitHub Pages | kostenlos |
| Kontaktformular | Formspree | kostenlos (bis 50/Monat) |
| Terminbuchung | Calendly | kostenlos (Basis) |
| Schriften | Google Fonts | kostenlos |
| OG-Bild | Canva | kostenlos |
| Favicon | favicon.io | kostenlos |
| SEO-Monitoring | Google Search Console | kostenlos |

### Google Fonts korrekt einbinden

**Immer als @import am Anfang der CSS-Datei — nicht als Kommentar!**

```css
@import url('https://fonts.googleapis.com/css2?family=Cormorant+Garamond:ital,wght@0,300;0,400;0,600;1,300&family=DM+Sans:wght@400;500;600&display=swap');
```

### Formspree einrichten

1. Account auf formspree.io erstellen
2. Neues Formular anlegen → Endpunkt-URL kopieren
3. Im HTML: `<form action="https://formspree.io/f/[ID]" method="POST">`
4. Honeypot gegen Spam: `<input type="text" name="_honeypot" style="display:none;" />`

### GitHub Pages Deployment

1. Repository erstellen (Name: `username.github.io` oder beliebig)
2. Alle Website-Dateien hochladen (index.html, style.css, main.js, Bilder/, etc.)
3. Settings → Pages → Branch: main → Save
4. Custom Domain: eigene Domain eintragen + CNAME-Datei im Repo
5. Bei Domain-Anbieter: CNAME-Eintrag auf `username.github.io` setzen

**Wichtig:** GitHub Pages ist Linux-basiert → Dateinamen sind case-sensitiv.
`Bilder/1.jpg` ≠ `bilder/1.jpg` → Immer einheitliche Schreibweise verwenden.

### Wichtige HTML-Grundstruktur (Template)

```html
<!DOCTYPE html>
<html lang="de">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>[Titel | Name – Positionierung]</title>
  <meta name="description" content="[140–160 Zeichen]" />
  <meta name="robots" content="index, follow" />
  <meta property="og:title" content="..." />
  <meta property="og:description" content="..." />
  <meta property="og:type" content="website" />
  <meta property="og:locale" content="de_DE" />
  <meta property="og:image" content="https://domain.de/og-bild.jpg" />
  <meta property="og:image:width" content="1200" />
  <meta property="og:image:height" content="630" />
  <link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png" />
  <link rel="stylesheet" href="style.css" />
</head>
<body>
  <!-- Navigation -->
  <!-- Hauptinhalt -->
  <!-- Footer -->
  <script src="main.js"></script>
</body>
</html>
```

## 9. Schritt-für-Schritt Systemanleitung

### Phase 1: Strategie (vor dem Computer)

**Schritt 1: Ziel & Zielgruppe definieren**
- Wer soll die Website besuchen?
- Was sollen sie dort tun?
- Was ist das eine Hauptziel?
- Ausgabe: 1 Satz Positionierung + 1 Satz Hauptziel

**Schritt 2: Positionierung & Kernbotschaft festlegen**
- Formel anwenden: „Ich helfe [Zielgruppe], [Problem] zu lösen, damit [Ergebnis]."
- 3 Hauptvorteile des Angebots formulieren
- Tonalität festlegen (Du/Sie, Stil)
- Ausgabe: Positionierungsaussage + 3 USPs

**Schritt 3: Seitenstruktur planen**
- Standard-Struktur verwenden und bei Bedarf anpassen
- Für jede Seite: Was ist der Zweck? Was ist der CTA?
- Ausgabe: Liste aller Seiten mit je einem Satz Zweck

---

### Phase 2: Vorbereitung

**Schritt 4: Inhalte sammeln**
- Fotos (mindestens 1 gutes Porträt)
- Alle Texte als Rohentwurf
- Logo (falls vorhanden)
- Alle rechtlichen Pflichtangaben (Impressum: Name, Adresse, E-Mail, Telefon)

**Schritt 5: Design-System festlegen**
- 2 Farben + 1 Neutralton wählen
- 2 Schriftarten auswählen (Google Fonts)
- Abstands- und Größen-Raster festlegen
- Design-System als CSS Custom Properties anlegen

**Schritt 6: Technisches Setup**
- Domain registrieren
- GitHub-Repository anlegen
- Formspree-Konto einrichten
- Calendly einrichten (falls Buchungen gewünscht)

---

### Phase 3: Umsetzung

**Schritt 7: HTML-Grundgerüst erstellen**
- Template für jede Seite anlegen (Head, Nav, Footer einheitlich)
- Gemeinsame Elemente (Nav, Footer) als Kopiervorlage
- Alle Seiten mit korrekten Meta-Tags anlegen

**Schritt 8: CSS-Design aufbauen**
- Google Fonts @import ganz oben
- Custom Properties definieren (Farben, Schriften, Abstände)
- Basisstile (Reset, Body, Container, Typografie)
- Komponenten (Nav, Buttons, Cards, Footer)
- Mobile-Media-Queries (maximal 2: 768px, 640px)

**Schritt 9: Inhalte einfügen**
- Texte nach AIDA-Prinzip strukturieren
- Bilder mit korrekten Alt-Texten und `loading="eager/lazy"`
- Alle internen Links prüfen
- Formular und Buchungslinks einbinden

**Schritt 10: SEO implementieren**
- Meta-Titel und Descriptions für alle Seiten
- H1 auf jeder Seite (genau eine)
- OG-Tags auf allen Seiten
- Sitemap erstellen
- robots.txt erstellen
- Favicon einbinden

**Schritt 11: Conversion-Elemente prüfen**
- Ist der CTA auf jeder Seite klar sichtbar?
- Gibt es einen primären und optionalen sekundären CTA?
- Ist der Prozess erklärt (was passiert nach Kontakt)?
- Gibt es Vertrauenselemente?

---

### Phase 4: Qualitätssicherung

**Schritt 12: Testen**
- Alle Seiten im Desktop-Browser testen
- Alle Seiten auf dem Smartphone testen (echtes Gerät)
- Alle internen Links klicken
- Formular absenden (Test-Nachricht)
- Buchungslink testen
- Navigation auf Mobile testen
- Bilder: Werden alle angezeigt? (case-sensitive Pfade!)

**Schritt 13: Inhalte finalisieren**
- Rechtschreibung und Grammatik prüfen
- Zeichenanzahl Meta-Texte prüfen
- Preise und Kontaktdaten verifizieren
- Impressum vollständig?

---

### Phase 5: Launch

**Schritt 14: Livegang**
- Alle Dateien auf GitHub hochladen
- GitHub Pages aktivieren
- Custom Domain eintragen
- DNS beim Domain-Anbieter konfigurieren
- HTTPS aktiv? (GitHub Pages: automatisch nach ~24h)
- Website über eigene Domain aufrufen und testen

**Schritt 15: Nach dem Launch**
- Google Search Console einrichten und verifizieren
- Sitemap einreichen
- Google Business Profil aktualisieren (URL eintragen)
- Website auf LinkedIn und anderen Profilen verlinken
- Erste Besucher auf die Seite aufmerksam machen

## 11. Übertragbares System

### Was immer gleich bleibt

**Struktur:**
- Startseite → Leistungen → Über mich → Kontakt → Impressum → Datenschutz
- AIDA-Aufbau auf jeder Seite
- Navigation oben, CTA in der Navigation, Footer mit allen Links

**Technisch:**
- HTML-Template mit vollständigem Head (Meta, OG, Favicon)
- CSS Custom Properties für Design-System
- Google Fonts via @import
- Sitemap + robots.txt
- Formspree für Kontaktformular
- GitHub Pages als Hosting

**SEO:**
- Meta-Titel nach Formel: `[Seitenthema] | [Name] – [Positionierung]`
- Description mit konkretem Nutzen und Handlungsaufforderung
- OG-Bild auf jeder Seite
- Google Search Console nach Launch

**Rechtlich:**
- Impressum mit allen Pflichtangaben
- Datenschutz mit allen verwendeten Diensten
- Formular mit Datenschutz-Checkbox

### Was flexibel angepasst wird

| Element | Anpassung nach |
|---|---|
| Farben & Schriften | Branche, Zielgruppe, persönlicher Stil |
| Tonalität | Du/Sie je nach Zielgruppe |
| Seitenanzahl | Komplexität des Angebots |
| Preistransparenz | Branchenüblichkeit und Strategie |
| Buchungstool | Ob Gespräche oder direkte Buchungen gewünscht |
| Bildstil | Persönlich, professionell, lifestylig |

### Das System in einem Satz

> Strategie zuerst → Design-System festlegen → HTML-Template aufsetzen → Inhalte nach AIDA einfügen → SEO-Checkliste abarbeiten → Launch → Search Console.

### Wiederverwendbare Datei-Pakete

```
template/
├── index.html          ← Head + Nav + Footer als Vorlage
├── style.css           ← Custom Properties + Basis-Stile
├── main.js             ← Mobile Nav + Scroll-Animationen
├── sitemap.xml         ← URLs anpassen
├── robots.txt          ← Domain anpassen
└── CHECKLISTE.md       ← Diese Checkliste
```

### Scroll-Animation (wiederverwendbar)

```css
.reveal {
  opacity: 0;
  transform: translateY(36px);
  transition: opacity 0.8s cubic-bezier(0.16, 1, 0.3, 1),
              transform 0.8s cubic-bezier(0.16, 1, 0.3, 1);
}
.reveal.revealed {
  opacity: 1;
  transform: translateY(0);
}
```

```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('revealed');
      observer.unobserve(entry.target);
    }
  });
}, { threshold: 0.1 });

document.querySelectorAll('.reveal').forEach(el => observer.observe(el));
```

## Anhang: Häufige Fehler und ihre Lösungen

| Fehler | Ursache | Lösung |
|---|---|---|
| Bilder werden nicht angezeigt | Falsche Groß-/Kleinschreibung im Pfad | Dateipfade case-sensitiv prüfen |
| Mobile Menü funktioniert nicht | Fehlende `id` am `<ul>` Element | `id="nav-links"` am ul-Tag prüfen |
| Grid bricht auf Mobile nicht um | Inline-Style überschreibt Media-Query | Inline-Styles entfernen, nur CSS-Klassen |
| Großer Leerraum im Hero | `min-height: 100vh` gesetzt | min-height entfernen, nur padding verwenden |
| Google Fonts laden nicht | Fehlender @import in CSS | @import ganz oben in style.css setzen |
| Formular kommt nicht an | Falsche Formspree-ID | Endpunkt-URL aus Formspree-Dashboard kopieren |
| Search Console Verifikation schlägt fehl | GitHub Pages noch nicht deployed | 5–10 Minuten warten, dann erneut versuchen |
| Sitemap 404 | Datei nicht hochgeladen | sitemap.xml in GitHub hochladen |
